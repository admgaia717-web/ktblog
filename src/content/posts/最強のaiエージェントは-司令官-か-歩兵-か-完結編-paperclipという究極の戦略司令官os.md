## Paperclipとは何か

公式の一言で完璧に表現されている：

*「Claude Codeが従業員なら、Paperclipは会社である」*

Paperclipは**AIエージェントのための指揮統制システム**だ。Node.jsサーバー＋React UIでできた、MITライセンスのOSS。2026年3月のローンチから数週間で36,000 GitHub Starsを獲得した。

## 何をするのか

**司令官の定義**

前回、AIコーディングエージェントを「司令官型」と「歩兵型」に分類した。司令官型（Claude Code、Codex、KiloCode、OpenCode）は巨大なシステムプロンプトでエージェントを「教育」する。

Paperclipはその一段上の司令官だ。**単一エージェントのプロンプトではなく、エージェントの組織そのものを管理する。**

``` [Paperclip] ← あなた（取締役会） │ ┌──────────┼──────────┐ │          │          │ [CEO]    [CTO]     [QA責任者] Sonnet   Sonnet      Haiku │ ├── [Backend Lead] (Sonnet) ├── [Frontend Lead] (Sonnet) ├── [テスト担当]     (Haiku) └── [DevOps]        (Haiku) ```

各エージェントには**役職・上司・予算・権限・作業指示書**がある。全員自律的に動き、定期的に「ハートビート（心拍）」で起床し、タスクを確認し、仕事をし、報告する。

## Paperclipの核心機能

**1. 組織図（Org Chart）**

エージェントは「従業員」として**役割・階層・レポートライン**を持つ。CEOがCTOに指示し、CTOが各チームリードに委任する——という構造を実際のAIエージェントで実現する。

**2. ハートビート（Heartbeat）——24時間自律稼働**

各エージェントは4/8/12時間ごとに自動起床。新鮮なコンテキストでタスクを確認し、作業を継続する。OpenClawの心拍パターンを継承した設計。**人間が寝ている間も、AIエージェントたちは働き続ける。**

**3. 予算管理（Budget Control）**

エージェントごとに月間予算を設定。80%で警告、100%で自動停止。暴走コストは物理的に発生しない。

**4. ガバナンス（Governance）**

あなたは「取締役会」。新規エージェントの雇用はあなたの承認が必要。CEOの戦略変更もあなたの承認が必要。全決定は監査証跡に記録される。

**5. チケットシステム**

全作業はIssueとして管理。依存関係、ブロッカー、ステータス、コメント。アトミックなチェックアウトで二重作業を防止。

**6. マルチカンパニー**

1つのPaperclipインスタンスで複数の「会社」を運営可能。データは完全分離。

## 対応エージェント（Adapter）

Paperclip v0.3.0（2026年3月）時点で標準対応：

エージェント | 種別

Claude Code | `claude_local`

OpenAI Codex | `codex_local`

Gemini CLI | `gemini_local`

Cursor | `cursor`

**Pi** | `pi_local`（RPCモード＋コスト追跡）

OpenCode | `opencode_local`

HTTP Webhook | カスタムエンドポイント

**PiもPaperclipから指揮できる。** 歩兵Piは単独でも最強だが、Paperclipの指揮下に入れば「組織の一兵士」として統制される。

## 司令官としてのPaperclip——なぜNo.1か

**1. 「司令官の中の司令官」**

Claude CodeやCodexは**戦術司令官**だ。一つのセッションの中でPlan/Implement/Testを指揮する。

Paperclipは**戦略司令官**だ。複数の戦術司令官を指揮下に置き、会社の目標に向けて統制する。

``` Paperclip（戦略司令官）： "今週の目標は認証システムのリファクタリング" ├── CEOエージェント → 作業分解 │   ├── Backend Lead（Claude Code）→ API実装 │   │   └── コード実装サブエージェント（Pi） │   ├── Frontend Lead（Cursor）→ UI改修 │   └── QA Lead（Codex）→ テスト自動化 └── 予算執行率 42%、監査証跡クリーン ```

**2. 20個の端末問題を解決**

*「同じプロジェクトでClaude Codeの端末を20個開いて、誰が何をやってるか把握できなくなる」*

これがPaperclipの直接の解決対象。**全エージェントの状態が1つのダッシュボードで見える。**

**3. 48エージェントのテンプレート**

GStack（CEO、CTO、QA、リリースエンジニア、スタッフエンジニア......）など、48エージェントの事前構築テンプレートがGitHubで公開されている。インポートするだけで大企業が立ち上がる。

**4. MITライセンス・完全セルフホスト**

Paperclip自身がOSS。Node.jsプロセス1つで起動し、組み込みPostgresで動作。外部依存ゼロ。アカウント不要。

## Paperclipではないもの

公式が明示的に「これは違う」と言っているもの：

- ❌ チャットボットUIではない

- ❌ n8nやZapierのようなドラッグ＆ドロップのワークフロービルダーではない

- ❌ エージェントを作成するフレームワークではない

- ❌ 単一エージェントのユースケース向けではない

*「たまに1つのAIエージェントを実行するだけなら、Paperclipは過剰。継続的な作業で3つ以上のエージェントを実行しているなら、Paperclipは欠けていたピース。」*

## 歩兵Pi × 司令官Paperclip——最強の組み合わせ

Pi単体は驚くほど軽量で高速。antirezが「saner」と評した通り、最小限の道具で最大の成果を出す歩兵だ。

しかし、**一人の歩兵では戦争に勝てない。**

PaperclipはPiを歩兵として、Claude Codeを将校として、Codexを砲兵として——多種多様なエージェントを一つの指揮系統に統合する。

あなたは取締役会会長として、その全軍を俯瞰する。

これが2026年の最強構成だ：

- **Pi**: 最速・最軽量の歩兵。個人作業はこれだけでいい

- **Paperclip**: 戦略司令官OS。複数エージェントで会社を回すときに投入

## 三部作の結論

| 歩兵 Pi | 司令官 Claude Code | 戦略司令官 Paperclip

役割 | 最速の実行者 | 戦術指揮官 | 会社の経営OS

システムプロンプト | 最小限 | 中程度 | なし（エージェントに任せる）

起動時間 | 数秒 | 中程度 | エージェント依存

向き | 個人開発者 | 単一プロジェクト | 自律的公司運営

エージェント数 | 1 | 1（+サブ） | 無制限

予算管理 | なし | 限定的 | 月額予算＋ハードリミット

ライセンス | MIT | Proprietary | MIT

**歩兵だけでも戦える。司令官がいればチームが組める。Paperclipがあれば会社が回る。**

これが2026年、AIエージェント設計思想の全スペクトルだ。
